**Research Completed:**

**Deno Deploy Storage Options:**

1. **Ephemeral Storage (/tmp)** - File system access
   - 10 GB ephemeral storage
   - Files deleted on restart
   - Worker-specific (not shared)
   - Easy API: `Deno.writeFile('/tmp/file.mp3', buffer)`

2. **Cache API** - HIGHLY RECOMMENDED ⭐
   - Supports **10-minute TTL** ✅
   - High performance (sub-millisecond latency)
   - Global distribution
   - FREE during beta
   - Worker-shared
   - Standard HTTP caching headers:
     - `Cache-Control: max-age=600` (10 minutes)
     - `Expires` header
   - Automatic cleanup

3. **KV Database** - NOT recommended for audio
   - 1 GB storage limit (during beta)
   - 64-256KB per key (too small for audio)
   - Good for metadata only

**Cache API Implementation Example:**
```typescript
const cache = await caches.open("agent-memo-audio");
const response = new Response(audioBuffer, {
  headers: {
    "Content-Type": "audio/mpeg",
    "Cache-Control": "max-age=600"  // 10 minutes
  }
});
await cache.put(new Request(`/audio/${memoId}`), response);

// Retrieve
const cached = await cache.match(new Request(`/audio/${memoId}`));
if (cached) return cached;
```

**Storage Capacity Analysis:**
- Peak concurrent files: 1,000 (100 req/min × 10 min)
- Peak storage: 1,000 × 200KB = 200 MB
- All Deno options handle this easily

**Migration Path:**
- Replace `fs.writeFile()` with `cache.put()`
- Convert `server.mjs` to `server.ts` (TypeScript)
- Replace `express` with `Deno.serve()`

## Technical Notes

### Deno Deploy vs Node.js Architecture

**Node.js/Express (Current):**
```
Request → Express → MemoService → ElevenLabs API → File System storage
                                                    ↓
                                                /storage/*.mp3
```

**Deno Deploy (New):**
```
Request → Deno.serve() → MemoService → ElevenLabs API → Cache API
                                                   ↓
                                                10-min TTL
```

**Key Difference:**
- Node.js: Persistent file storage (`/storage/` directory)
- Deno: Cache API with automatic expiration

### ElevenLabs Authentication
The API requires specific header format that differs from standard OAuth:
- **Standard OAuth:** `Authorization: Bearer {token}`
- **ElevenLabs:** `xi-api-key: {api_key}`

This is documented at https://elevenlabs.io/docs but not obvious from their code examples.

### Deno Deploy Cache API

**Cache Control Headers:**
```
Cache-Control: max-age=600        # 10 minutes (600 seconds)
Expires: Thu, 13 Feb 2026 18:30:00 GMT
```

**Cache Behavior:**
- First request: generates audio, caches for 10 min
- Within 10 min: serves from cache (sub-ms latency)
- After 10 min: cache expires, generates again

**Cache Storage:**
- Distributed across edge locations
- 512KB pages, multi-tier storage (RAM → SSD → Cloud)
- 2.4Gbps throughput, sub-millisecond latency
- FREE during beta

### Cache API 10-minute TTL
Deno Deploy's Cache API supports standard HTTP caching headers:
- Use `Cache-Control: max-age=600` for 10 minutes
- Automatic expiration and cleanup
- Files served from edge locations for low latency

### File Size Differences
The audio quality directly impacts file size:
- **Simulation:** 10KB - minimal data (essentially silence)
- **Edge TTS:** ~50KB estimated
- **ElevenLabs:** 200-230KB - full high-quality speech

This confirms that real ElevenLabs audio generation is working correctly.
